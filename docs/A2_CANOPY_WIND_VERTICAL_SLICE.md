# A2「林冠没有风」可玩纵切

日期：2026-07-15
状态：已实现并通过权威模拟、存档、交互、音频、UI 与生产包验收；A3–A5 仍属后续里程碑
目标时长：首次玩家 35–45 分钟，熟练玩家 22–30 分钟

## 1. 纵切承诺

A1 上报“河水持续上涨”后，应急网络返回一条矛盾记录：主雨带正在推进，但林冠风速通道连续数小时为 `0.0`。玩家不是去捡一个任务零件，而是要亲自证明“雨林里有风、记录里没有风”，找到被倒木与藤本拉扯的林下接线盒，恢复传感链路，并用一次真实阵风完成校验。

玩家应感到：

- 叶片、雨线、藤蔓和声音是可以用于判断的环境信息，不只是装饰。
- 准备方式会改变远征风险，但不会锁死唯一解法。
- 建在路上的遮雨点、集水架和加工设施在任务后仍然有用。
- 修复改变的是一处持久世界状态；退出、死亡、读档和提前发现都不会把它重置成脚本开关。

本纵切不新增完整电工小游戏、复杂电路元件、独占任务材料、新动物物种或架高前哨整套模块。第一版用现有斧/石刃、叶棚、床、营火、烟熏架和雨水收集架证明闭环；架高前哨留给 A3。

## 2. 原创叙事节点

自动回执原文意图：

> “河谷水位包已收到。林冠通道仍报告静风，但上行雨带不可能在绝对静风中推进。请核验 C-17 林下汇线箱；不要把零值当成天气。”

C-17 原本把高处的机械风杯信号汇入埋地线路。暴雨前的一次倒木没有砸毁设备，而是慢慢拉紧攀附在电缆上的藤本，使防水插头退出半圈。载波灯仍亮，所以远端把“没有脉冲”错误解释成“没有风”。这延续《失联河谷》的主题：雨林没有恶意，多个被忽视的小因果会在暴雨中放大成盲区。

接线盒是唯一叙事地标，但倒木、藤本、叶棚、火源、集水和加工全部遵守普通世界规则。不得借用《绿色地狱》或其他作品的角色、地点、文本、地图或任务表达。

## 3. 事实图与任务定义

### 3.1 稳定事实 ID

| 类型 | 事实 | 产生条件 |
| --- | --- | --- |
| `heard` | `radio.emergency-canopy-zero` | A1 水位上报后收到延迟回执；若玩家已发现或修好 C-17，使用对应变体 |
| `prepared` | `canopy-expedition.repair-kit` | 在离开安全设施后，以斧/石刃路线第一次处理遮挡；事实承认实际方法，不提交物品 |
| `prepared` | `canopy-expedition.provisioned` | 离开主营地/前哨时携带实体火把、至少 2 份净水，以及烟熏肉/熟肉 1 份或芭蕉果 2 份 |
| `prepared` | `canopy-expedition.forward-outpost` | 在 C-17 路线 32m 内形成一个合格设施组：叶棚 +（床、燃火营火、集水架或烟熏架任选其一） |
| `observed` | `canopy.wind-present-feed-zero` | 实际风强度可读时调查 C-17：叶片/雨线在动，但面板仍显示 `0.0` |
| `changedWorld` | `canopy-junction.link-restored` | 遮挡解除、防水插头复位并锁紧，状态写入地标差量 |
| `observed` | `canopy-junction.live-sample` | 修复后完成一段有效阵风采样，并由玩家查看方向、强度与信号质量 |
| `reported` | `canopy.wind-sample` | 在已供电的求救信标上发送该有效样本；允许提前主动上报 |

任务完成条件是：

```text
heard
AND prepared(repair-kit OR provisioned OR forward-outpost)
AND observed(wind-present-feed-zero)
AND changedWorld(link-restored)
AND observed(live-sample)
AND reported(wind-sample)
```

`prepared` 不是一道事前门禁。玩家可以先发现、先修复，再由实际采用的修理路线、补给或设施方案获得相应事实。UI 不展示隐藏布尔清单，只展示当前尚未理解的世界问题。

### 3.2 提前发现契约

- C-17、遮挡树、藤本和面板从新游戏开始就存在并可自然交互，不因 A2 尚未激活而失效。
- 提前调查时记录 `observed` 和笔记；提前清障、修复、采样、上报均照常保存事实。
- A2 回执到达后按已有事实跳过相应引导，并改为“你找到的 C-17 正是失联通道”。
- 若玩家在 A2 前已全部完成，回执确认其主动报告，任务直接结算，但不重复奖励、扣料或重建地标。
- 真正不能操作时只给世界内原因：工具效率不足、倒木仍压住箱门、暴雨正灌入开口或插头尚未锁紧；禁止“任务阶段未到”。

## 4. 空间、路线与 35–45 分钟节拍

C-17 使用一个稳定 authored ID，初始候选位置为营地东北约 150m 的密林—芭蕉扰动带。落点必须通过自动探针确认：不在河面、坡度可行、距主路线 8–18m、32m 内同时存在一处较开阔集水点和一处天然遮雨选址。坐标探针失败时调整 authored 常量，不能运行时随机换点。

至少保留两条可读路线：

- **湿谷捷径**：芭蕉和断藤多，补给容易，地面更湿且视距短。
- **坡肩绕路**：更长、更干、视野较好，石料与路标选址更清楚。

| 分钟 | 玩家行为 | 必须产生的反馈/结果 |
| ---: | --- | --- |
| 0–3 | 收到矛盾回执，查看地图/笔记 | 电台波形有载波无脉冲；任务指向“核验零值”，不直接给修理答案 |
| 3–9 | 选择补给、工具或先建路线设施 | 三种 `prepared` 方案至少一条可成立；配方只作支持，不强制收集同一种材料 |
| 9–19 | 走湿谷或坡肩，经过芭蕉带 | 风向通过叶面翻转、雨线倾斜和音景可读；路线风险与资源不同 |
| 19–27 | 找到 C-17，比较真实风与零值，解除遮挡 | 取得矛盾 `observed`；倒木/藤本发生可见、可保存变化 |
| 27–33 | 复位插头、锁防水帽，开始采样 | `changedWorld`；面板从零值变为“等待有效阵风”，线路指示灯恢复脉冲 |
| 33–38 | 等候自然天气窗口并经营前哨 | 集水、烹饪/烟熏、添柴、整理负重或记录植物；当前已有阵风则可更快完成 |
| 38–45 | 查看样本，返营或沿已建路线到信标上报 | `observed(live-sample)`、`reported`、任务里程碑检查点；世界继续运行 |

不得通过要求 20 根同类材料填时长。单一路线连续同类采集不应超过 8 分钟；等待可读阵风的实际时间上限为 4 个现实分钟。

## 5. C-17 世界交互

### 5.1 可读模型

- 0.9m 高的黄色防水箱立在两根桩上，门、锁扣、橙色 `C-17` 编号和向上延伸的电缆均可看清；不得做成岩石或山包轮廓。
- 倒木是普通语义树的倒伏状态，藤本是普通可切植物；相同模型在别处提供相同基础动词。
- 面板有三种世界状态：稳定载波/零脉冲、接头开放、真实脉冲采样。UI 仅解释，不替代灯光、指针和声音。

### 5.2 状态机

```text
obstructed
  -> exposed
  -> connector-open
  -> link-restored
  -> sampling
  -> sample-ready
  -> reported
```

- **解除遮挡**提供两条同规则路径：斧击倒木的受力点；或用石刃切断两处受拉藤本后，以 4 根木棍 + 2 根藤条搭临时杠杆。两者都通过动作命中与二次校验结算。
- **打开接头**时若雨势较强且箱体不在实际叶棚覆盖内，操作被阻止并明确提示“开盖会进水；搭遮雨棚或等雨势减弱”。任务永远保留等待天气的零材料方案。
- **复位**是一次 2–3 秒的具身操作：拔出、对齐标记、压入、旋紧防水环；中断不吞材料、不半写事实。
- **采样**由设备自行持续。修复时若已有可读阵风，10 秒稳定窗后完成；若当前静风，天气计划保证 4 个现实分钟内出现一个可读窗口。玩家离开活动泡后以低频确定性聚合，不要求盯着进度条。
- 样本至少显示方向扇区、归一化风强和信号质量，不伪装成科学级精确测量。

## 6. 风与叶片的同一真相

新增的权威风场属于确定性天气状态，而不是渲染器时钟：`directionRadians`、`speed`、`gust`、目标值与下一次前线时刻均由世界种子、固定 tick 和天气前线推进。C-17 故障只让**上报值**为零，不能把真实世界风设为零。

表现分四档：

| 风档 | 世界反馈 |
| --- | --- |
| 静风 `<0.18` | 叶片下垂、雨线近垂直、音景闭合；面板可诚实提示暂无法校验 |
| 微风 `0.18–0.45` | 藤梢偏向下风，蕨叶连续摆动，芭蕉叶偶尔露出浅色背面 |
| 强风 `0.45–0.72` | 大叶明显翻折、雨线倾斜、树冠方向一致、枝叶声增强 |
| 阵风 `>0.72` | 冠层波动、少量落叶沿风向移动、木架轻响；不靠屏幕震动表达 |

实现约束：

- L1 语义植物、L2 地被、L3 中景叶团共享同一个风向与强度，只允许由稳定实体 ID 产生相位差。
- C-17 周围设置 6–10 个“可读指示叶”，它们仍是普通芭蕉/阔叶表现，不是隐藏任务对象。
- 低画质和移动端可以减少叶片数量，不能删除风向信息；至少保留指示叶、雨线倾斜和方向性音景三者中的两者。
- `prefers-reduced-motion` 降低摆幅与频率，但保留叶背明暗变化、雨线方向和调查文本，不把可访问性变成玩法劣势。
- 粒子、叶片和声音只投影风场；不得反向决定任务事实。

## 7. 前哨与集水架为什么有用

A2 不要求在脚本点放唯一建筑，而是检查玩家自由放置的真实实例：

- 叶棚允许暴雨中安全打开接线盒，也降低等待时湿度增长。
- 雨水收集架在林隙效率更高，给返程与之后 A3 的湿地路线留下稳定补水点。
- 营火帮助烘干并提供夜间返程锚点；烟熏架把等待窗口转成远征口粮加工。
- 床允许玩家在自选前哨休息并触发既有休息前/后检查点，但时间、饥渴、腐败、天气和火继续推进。

一个“合格前哨”只要求叶棚加任一配套设施，不限制实例数、朝向或固定坐标。设施效果查询具体结构 ID 与真实几何范围；任务完成后不拆除、不返还成抽象奖励。

## 8. 失败、离开与恢复

- **死亡**：进入既有死亡复盘与检查点时间线；推荐最近的休息前、A2 出发或修复完成检查点，不自动新开局。
- **任务检查点**：首次获得 `heard`、`changedWorld(link-restored)` 和 `reported` 时各写一个旋转自动档；休息继续写 rest-before/rest-after。
- **工具损坏**：保留石刃切藤 + 杠杆路线；资源导演只能安排合法到期节点，不能在脚边生成新斧头。
- **大雨**：叶棚与等雨减弱均可恢复；不得让接头在开放状态永久损坏。
- **静风**：显示下一天气窗的环境征兆；确定性上限防止软锁，但不在玩家身边凭空生成“任务阵风”。
- **背包满**：清障产出留在地面或原实体库存，链路事实仍能原子提交。
- **离开/卸载**：接线盒、遮挡、采样和设施按差量保存；采样低频推进，重进活动泡不重置倒计时。
- **损坏存档**：单槽隔离；玩家可从时间线选择更早检查点。任务 UI 不把本地未落盘状态说成“已保存到云端”。

## 9. 资源导演支持边界

A2 的支持配方为斧、石刃、绷带、火把、叶棚、雨水收集架和烟熏架。Director 只用它们计算需求排序：

- 只从已经到期、生态合法的藤条、木棍、宽叶、石块、药草和食物节点中每 epoch 选择至多一个。
- 继续遵守现有 48m 保护距离、背后/卸载优先、容量、刷新窗口和确定性 seed；树木、矿体、稀有物和任务地标不进入快速恢复池。
- 湿谷提高藤条、宽叶与芭蕉候选适合度，坡肩提高干叶、木棍与石块适合度；玩家缺什么不能改变物种或突破群系。
- 两条 authored 路线各保证一次基础可行性，不保证一键毕业：至少能找到一种修理路径所需的普通材料，但不在 C-17 门口堆成补给箱。
- 丢弃/拾取物品不能重滚 Director；存档恢复后 evaluated epoch 不倒退。

## 10. UI、音频与输入

- 主任务卡只显示当前问题与下一条可行动建议；笔记保存所有已知事实、实测值和提前发现记录。
- 聚焦叶片给出克制的“受风方向”调查信息；聚焦 C-17 显示真实物理状态、主动作和阻塞原因。
- 修复、锁环、脉冲恢复和采样完成均先有模型、声音和灯光反馈，再出现短消息。
- 键鼠和触控拥有相同动词：调查、清障、复位、查看样本、使用设施、上报。移动端不新增常驻大面板，世界交互按钮复用现有上下文入口。
- 风声使用方向性混合，但关键事实同时有视觉与文字渠道；不得只靠立体声辨向。

## 11. 性能与存档预算

- 风场每固定 tick 更新权威值；可见叶片使用共享 uniform/实例相位，不为每片叶建立 React 状态或独立模拟实体。
- C-17 及其状态表现新增稳定 draw submission 目标不超过 4；不新增实时阴影灯。
- 风向落叶上限：桌面 48、移动端 16；对象池复用、无碰撞、离屏停止表现。
- 移动低画质的风指示叶不少于 4 个；L2/L3 可降密度，L1 任务与资源对象不可删除。
- UI 投影最高 4Hz；无每帧存档写入。C-17 全部差量与事实预计控制在 2KB JSON 以内。
- 固定 A2 路线对比同构建基线，目标新增 CPU p95 不超过 1.5ms（桌面）/2.0ms（目标移动档）；必须记录实际浏览器数据，未测不得宣称通过。

## 12. 自动化验收

1. 同 seed、tick 和天气输入得到相同风向、强度、阵风窗和 C-17 样本；无 NaN、越界或存读漂移。
2. 故障前真实风可非零但上报固定为零；修复后上报只来自同一权威风场。
3. 六类任务事实顺序无关、幂等、保留 earliest tick；三种 `prepared` 为 OR，其余为 AND。
4. 提前发现、提前修复、提前上报后再收到 `heard`，任务能正确完成且不重复结算。
5. 两条清障路径都经过预检—动作—二次校验—原子结算；中断、工具损坏、材料变化均无负库存或半事实。
6. 暴雨无遮蔽时开盖被明确阻止；真实叶棚覆盖或雨势下降后可继续，且判定使用共享结构几何。
7. 修复后的有效阵风最迟在设计上限内到达；离开活动泡、休息、保存/加载后采样进度确定性一致。
8. 任意多实例叶棚、床、营火、集水架和烟熏架中，只有满足距离与局部状态的实例构成前哨。
9. Director 不刷新树、矿、稀有物或任务实体，不在 48m 内/前方结算，不因丢弃物品重滚。
10. `changedWorld` 与 `reported` 分别产生任务自动档；死亡状态不能覆盖唯一安全恢复点。
11. A2 完成只推进章节，不把 `GameState.status` 改成胜利或冻结沙盒。

## 13. 浏览器试玩闸门

- 5 名首次测试者中至少 4 名在不读教程时，能在 5 秒内从叶片/雨线判断风来自哪个 90° 扇区，并能解释“世界有风但记录是零”。
- 首次完成中位数 35–45 分钟；没有连续超过 8 分钟只采同一种材料，没有超过 4 分钟无可做事项的强制等待。
- 三种准备方案分别至少完成一次：工具快修、补给远征、前哨经营；任何一种都不要求唯一脚本建筑位。
- 从两条路线各完成一次；湿谷与坡肩在湿度、视距、资源和耗时上有可描述差异，但都无设施也可达。
- 提前发现测试能自然调查并修复 C-17，不出现“阶段未到”；之后无线电文案与目标状态正确变体。
- 在强雨、静风、工具破损、背包满和途中死亡场景中均能恢复，不新开局、不丢失已验证检查点。
- 桌面 1920×1080、移动 390×844/844×390 以及 reduced-motion 各跑完整闭环；核心按钮不遮准星，风事实至少有两个感知渠道。
- 保存、重载、离开区块再返回后，遮挡、插头、样本、设施与任务事实一致；Toy 云未完成时 UI 明确显示本地状态。
- 按既有性能协议记录 p50/p95/p99、draw、triangle、活动对象数、内存和跨区尖峰；未记录数据不得跨过 A2 闸门。

## 14. 实施切片顺序

1. **A2-0 事实与内容**：稳定 ID、任务图、无线电变体、提前发现/上报测试。
2. **A2-1 风的同一真相**：确定性风场、雨线/叶片/音景投影、低画质与 reduced-motion 可读性。
3. **A2-2 C-17 交互**：地标几何、遮挡、两条清障路径、接头状态机、差量存档。
4. **A2-3 前哨中循环**：三种准备事实、多实例设施资格、采样等待中的加工/集水、Director 支持。
5. **A2-4 集成闸门**：任务检查点、死亡恢复、键鼠/触控、浏览器试玩和性能记录。

每一步都要先通过自动化与一个可玩的最薄路径再扩宽。A2-1 的叶片风向若不能被玩家读懂，禁止用更多任务文案掩盖；A2-3 的等待若不能自然转化为营地行为，应缩短采样窗，而不是增加材料配额。
