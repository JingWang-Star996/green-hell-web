# CANOPY：世界对象与交互歧义审计

日期：2026-07-14
性质：代码只读审计；尚未替代实机试玩

## 总结

当前根因是“视觉对象系统”和“玩法对象系统”分裂：

- RenderEntity 只有 interactive / available 二值状态。
- 建筑、生态动物、区块装饰和可交互实体分别走不同投影/渲染路径。
- 目标解析只遍历 entityViews。
- 输入动画、模拟命令和 UI 反馈又各自判断一次动作。

因此玩家看到一个对象，并不能稳定推导它能否交互、需要什么、会发生什么；同一个动作从 E、鼠标或触屏进入时也可能有不同表现。

## P0：先修才能建立可信世界

### P0-A 统一 affordance

玩家预期：

- 准星指向对象后知道它是什么、能做什么、缺什么、风险是什么。

现状：

- 提示动词主要由 kind 猜测。
- interactive 是二值。
- 无 blocker、工具要求、遮挡检测或对象高亮。

最小改造：

- ready / blocked / depleted / ambient / danger 五态。
- ResolvedAffordance 统一提供 action、verb、requiredItem、blocker、range、preview、highlight、animation 和 feedback。
- selector 预演已知条件，simulation 继续最终校验。

### P0-B 树、岩石与植物同形同规则

用户要求比“给可采对象做标记”更严格：

- 所有离散树都能被处理，区别只在工具、耗时、耐久和产出。
- 所有离散岩体都能开采；只有连续山体/岩壁可归为地形。
- 可收割植物拥有独特物种轮廓；微型地被显式 never-focus。

语义区块生成器必须成为模拟和渲染的共同来源。

### P0-C 水

- 活动区所有可接近河段应产生无库存的确定性 water-source affordance。
- 水质由群系、天气和环境决定。
- 提前显示容器/污染要求。
- 不持久化“水面库存”，避免无意义存档增长。

### P0-D 动作蛇

现状：

- 6.8m 警告。
- 1.45m 自动调用 encounter。
- 蛇本身 interactive=false。
- 装备矛只会在自动结算中改变结果。

目标：

- danger affordance。
- 盘伏、警告、瞄准、扑咬、恢复。
- 主动刺击、退让和绕行。
- 近身自动触发只作为攻击命中兜底，不是整个战斗。

### P0-E 建筑进入世界交互

- 营火：添柴/引燃/查看燃料。
- 床：休息。
- 叶棚：查看遮雨状态。
- 信标：修复/查看/发报。
- 菜单保留为二级详情，不再是唯一入口。

### P0-F 输入、动画和反馈单路

- E、鼠标、触屏都调用 performResolvedAction。
- animationKey 来自动作，不来自“手里随便拿着什么”。
- 命令结果通过 command/action ID 进入统一反馈通道。
- 世界 HUD、背包、身体、制作和建筑面板都显示同一次结算，不抓取无关的最新事件。

## P1：在可信交互后扩展

### P1-A 生态动物

当前物种有种群和视觉活动，但没有运行时个体后果。完整升级需要：

- 活动泡个体生命/感知/状态。
- 猎物逃逸、捕食者攻击、死亡、尸体和战利品。
- 离开活动泡后回写区块种群账本。

在完整切片前，任何尚不可交互的动物都应明确为 ambient，不能发出没有后续结果的假危险警告。

### P1-B 地标与电池状态

- 已调查地标仍可重读，状态从 new 变 recorded，而不是突然失去交互。
- 电池提前列出营地、线索、工具等已知前置条件。
- 柜门、电池和信标灯显示状态。

### P1-C 资源生命周期表现

- full / low / depleted / regrowing。
- 多单位节点逐渐减少。
- 树桩、空腐木、采空岩体或明确残迹。
- 后台再生不用远程 toast，玩家返回时能从世界读出状态。

### P1-D 建造一致性

- 放置失败返回坡度、水、树、建筑重叠、半径等 reason code。
- 信标要么进入统一放置协议，要么明确为“只能在电台定点修复”，不再伪装成自由建筑。

### P1-E 背包

- 材料、食物/水、医疗、工具和任务分节。
- 快捷装备与库存分离。
- 紧急上下文提供净水/绷带等快捷入口，但不替玩家自动做生存决策。

## P2：合理的非交互层

鸟群、萤火虫、普通地被和细碎表面颗粒可以不交互，但必须：

- 显式 classified ambient / never-focus。
- 不触发准星、动作提示或危险警告。
- 不复用离散资源的轮廓、尺度和材质。

## 当前最小安全切片

1. 新增纯 resolveAffordance selector。
2. 建筑、水源和蛇进入统一对象投影。
3. 增加遮挡、五态高亮和具体阻塞提示。
4. E、鼠标和触屏统一动作入口。
5. 所有面板共享动作反馈。
6. 语义区块计划接入模拟与渲染，替换树/岩/植物双轨。
7. 用测试覆盖每个核心对象的 ready/blocked/depleted/danger，并以实机对象扫查收口。

## 审计限制

本报告来自代码路径审计，能够证明系统分裂和规则冲突，但不能替代：

- 实机对遮挡、点击容错、动画节奏和视觉显著性的观察。
- 真实玩家首次理解率。
- 低端浏览器性能数据。
