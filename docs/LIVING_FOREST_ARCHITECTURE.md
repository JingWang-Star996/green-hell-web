# Living Forest：长期世界升级架构

> 目标不是把当前十几分钟流程拉长，而是让观察、判断、远征、加工、建设与生态变化反复产生新局面。主线可以简单，世界循环不能是一次性的。

## 1. 三层循环

### 小循环：20–90 秒

1. 通过声音、足迹、天气、植物状态或身体症状获得信号。
2. 判断当前风险与机会。
3. 选择采集、绕行、等待、驱赶、治疗或消耗工具。
4. 世界和身体立即反馈，玩家获得新的事实或路线信息。

每个动作至少改变两个系统。例如砍藤条不仅增加材料，还制造声音、消耗斧耐久，并可能惊动附近动物。

### 中循环：8–25 分钟

营地诊断与配装 → 选择路线和生态区 → 远征 → 因天气、动物或负重改变计划 → 返回加工、储存、维修与升级。

中循环的价值来自准备质量和途中应变，而不是反复拾取同一种资源。

### 大循环：45–120 分钟

建立前哨和安全走廊 → 解锁新的生态区与穿越能力 → 完成一组调查、修复或生存章节 → 世界进入新的气候与生态阶段。

完成主线后保留沙盒，不再以一个 `won` 状态永久终止世界。

## 2. 十小时节奏目标

| 时段 | 世界阶段 | 新约束与能力 |
| --- | --- | --- |
| 0–0.5h | 首夜教学 | 身体诊断、安全水、火、遮蔽和可读任务步骤 |
| 0.5–2h | 河谷活动 | 水食循环、腐败、第一种猎物、动态蛇、营地储存 |
| 2–4h | 岩岭远征 | 第二营地、工具耐久、高地缺水与风暴暴露 |
| 4–7h | 沼泽调查 | 寄生虫、泥泞、追踪、熏制与生态压力 |
| 7–10h | 风暴网络 | 多前哨、通信节点、气候阶段和跨区运输 |

## 3. 单一世界真相

当前模拟和渲染分别维护坐标、地形与碰撞，无法扩展。目标边界如下：

- `world/generation`：纯函数区块生成；输入世界种子、生成器版本和区块坐标。
- `world/runtime`：活动区块、空间索引与统一 `WorldQuery`。
- `world/resources`：资源容量、再生、生态压力和惰性时间结算。
- `ecology`：远处种群聚合与近处动物个体状态。
- `quests`：事实库、任务步骤和清晰的可执行动作/阻塞原因。
- `structures`：可放置、可维护、可储存的多个营地与前哨。
- `render/world`：只消费区块和实体差量，不自行发明另一套地形。
- `persistence`：保存种子和被修改过的区块差量，不保存可重建的基础世界。

基础稳定 ID：`generatorVersion:chunkX:chunkZ:spawnIndex`。

## 4. 功能生态区

生态区必须同时影响地形、视野、声音、移动、资源、动物与天气响应，不能只换颜色。

- 常绿密林：木材、藤条和药草丰富；视野短，声音传播近。
- 河谷湿地：水和鱼稳定；洪水、寄生虫和泥泞风险高。
- 棕榈坚果林：食物与动物交通密集；更容易暴露行踪。
- 黑水沼泽：稀有纤维和驱虫药；移动慢，疾病压力大。
- 岩石高地：石材、干燥庇护和远视野；水少，雷暴暴露高。

## 5. 资源经济

资源再生必须与消耗项同时存在，否则长期世界会通胀。

- 落枝和干叶：暴雨或大风后补充。
- 药草、藤条：按游戏日生长，过度采集降低恢复速度。
- 果实和坚果：受季节与动物取食影响。
- 石块：不凭空生长，由河流冲刷或崩塌暴露新节点。
- 任务电池与唯一剧情物件：永不再生。

对应消耗：食物腐败、工具耐久、燃料、建筑维护和动物取食。

再生只在节点离开玩家视野或区块卸载时显现，避免资源在眼前跳出。

## 6. 动物生态

动物分两层模拟：

- 环境层：鸟、蛙、蝶和萤火虫，以声音和活动提示时间、雨势与生态健康。
- 玩法层：小型猎物、蛇、大型食草动物与捕食者，具有觅食、饮水、休息、警戒、逃跑和追踪状态。

活动区块内实例化动物个体；远处只保存种群数量、承载力与食水压力。加载和卸载区块时在两者之间确定性转换。

危险必须先产生可读信号，再造成伤害。动物不能在玩家近处无提示生成。

## 7. 存档与性能

长期存档只保存：

- `WorldManifest`：种子、生成器版本、时间和探索区块。
- `ChunkDelta`：采集、结构、容器、生态和发现变化。
- `KnowledgeState`：永久事实、已知配方、物种和地标。
- `QuestRun`：当前步骤、计数与完成状态。
- 玩家、营地和物品状态。

知识不能继续从最多 256 条的事件日志反推；日志只用于近期反馈。

渲染只保留邻近区块，模拟只运行活动区块。十小时存档大小只随“被修改的区块”增长。

## 8. 交付阶段

### Phase A：Living Forest Foundation

- 修复导航、显式任务子步骤、交互优先级和岩棚。
- 普通资源惰性再生，任务物件保持唯一。
- 环境动物、动态蛇和一种小型猎物。
- 区块坐标、稳定种子、五类生态区数据与活动环。

### Phase B：Expedition Economy

- 三种生态区正式接入地形、资源和动物。
- 工具耐久、腐败、储物架、雨水收集器和熏制架。
- 第二营地与往返补给。

### Phase C：Streaming World

- 按需生成和卸载区块。
- 地图迷雾、跨区保存、远处种群聚合。
- 两公里连续穿越，无固定世界边墙。

### Phase D：Ten-hour Campaign

- 五生态区、多个前哨和通信调查网络。
- 天气前线、生态压力和章节目标池。
- 10 小时 soak、存档增长、资源经济和移动端性能调优。

## 9. 硬验收

- 玩家在 25 米外能认出岩棚入口，无需穿模即可看到并取得箱子。
- 罗盘、地图、任务方位和坐标使用同一方向约定。
- 任一任务阻塞都明确显示缺失对象、工具、位置与已完成进度。
- 资源按确定性时间再生，不提前、不越过容量、不在眼前出现。
- 动物先警戒再伤害，离开区块和读档后仍保持确定性。
- 跨区保存后修改仍在，未修改区块可由种子重建。
- 一百万 tick 长期模拟不丢知识、不产生 NaN，存档体积不随基础世界线性膨胀。
