# CANOPY：活体雨林执行队列

日期：2026-07-14
依据：docs/LIVING_RAINFOREST_GAMEPLAY_SPEC.md

## 1. 排序原则

1. 先消除世界规则歧义，再增加内容。
2. 先建立一个完整的聚焦—动作—反馈—持久化切片，再横向复制到更多对象。
3. 动物先有真实交互，再扩物种数量。
4. 建筑先形成群系问题闭环，再扩配方数量。
5. 玩法闸门通过后，才执行 Valheim 启发的程序化视觉重构。
6. GitHub、Toy 和外部通知保持暂停，直到本地可玩闸门通过并由用户恢复发布。

## 2. 依赖顺序

    S0 规格与对象审计
      ├─ S1 差量存档基线
      └─ S2 统一语义生成
            ├─ S3 世界对象/能力协议
            │    ├─ S4 聚焦与动作结算
            │    ├─ S5 树木与岩石切片
            │    └─ S6 篝火/背包/夜间体验
            └─ S7 活动泡与动物个体
                  └─ S8 动作蛇 + 猎物 + 捕食者
                         └─ S9 建造—群系—材料闭环
                                └─ V1 程序化视觉世界

S1 与 S2 可以并行；S6 可在 S4 稳定后与 S5 并行；V1 的研究可以并行，代码实现必须等待 S5/S8/S9 的语义输入稳定。

## 3. 任务包

### S0 — 对象真相审计

交付：

- 枚举所有渲染器生成、模拟器生成和 UI 伪装的对象。
- 标记同形不同规则、不可见热点、距离自动结算、菜单替代世界反馈和无法持久化状态。
- 为每一类对象决定：语义实体、连续地形、微型地被或纯特效。

主要检查位置：

- src/game/world/generation.ts
- src/game/world/harvestGeneration.ts
- src/game/render/RainforestRenderer.ts
- src/game/sim/content.ts
- src/game/sim/types.ts
- src/game/sim/simulation.ts
- src/game/ecology
- src/game/ui

完成闸门：不存在“尚未分类的可见离散模型”。

### S1 — 可扩展差量存档

交付：

- 世界种子与生成版本重建未改变区块。
- 只保存耗尽、受损、部分采集、死亡、放置、拆除和任务改变。
- 旧存档迁移后不会把所有访问过的基线实体重新写回。
- 模拟十小时长距离探索仍低于 Toy 128 键与每键值总计 1024 字节的约束。

完成闸门：访问更多未改变区块不会线性增加主存档。

### S2 — 单一语义区块生成

交付：

- 用一个生成计划产出树、岩石、资源植物、微型地被和水/地形信息。
- 离散树与岩石拥有稳定 ID、物种/材质、尺寸和能力定义。
- 渲染密度降级只改变网格/LOD，不改变对象是否真实。
- 删除或封闭现有独立的 visual trees/rocks 与 standing trees 双轨入口。

完成闸门：给定世界种子和区块坐标，模拟与渲染得到完全相同的离散对象 ID 集。

### S3 — 对象组件与能力解析

建议的最小组件：

- identity / transform / morphology
- harvestable / damageable / inventory
- toolRequirement / yieldTable
- lifecycle / regeneration
- focusPresentation / collision
- persistencePolicy

建议的最小动作：

- pickup / chop / mine / cut / attack / inspect / maintain / dismantle

交付：

- capability resolver 返回主动作、工具要求、阻塞原因、动作时长和反馈配置。
- 旧的 pick-up、harvest、encounter-hazard 命令逐步适配统一解析，不在第一步强行重写全部任务逻辑。
- 模拟器仍是结算权威，渲染器只提交目标 ID、动作和命中证据。

完成闸门：同一目标在 UI、输入和模拟中不再各写一套条件判断。

### S4 — 聚焦、命中和反馈

交付：

- 距离 + 视线 + 准星锥 + 遮挡的唯一聚焦目标。
- 近距轮廓/边缘光；微型地被不进入候选。
- 主动作、工具/阻塞原因和可选长按动作提示。
- 工具动作有命中窗口；射线/碰撞未命中就不结算。
- 材质音效、碎屑、反冲、目标状态变化和短 UI 反馈。

完成闸门：测试者不靠试按即可正确预测九成抽查对象的基础动作。

### S5 — 树木与岩石纵切

树木交付：

- 至少三种树材/生长级别。
- 所有离散树可被适当工具处理；错误工具有原因反馈。
- 受击、倒伏/替代、树桩、木料掉落、再生与差量存档。

岩石交付：

- 松散河石、建筑石岩体、燧石结核、黏土/红土露头的首版规则。
- 所有离散岩体可开采；连续岩壁不冒充节点。
- 裂隙命中节奏、工具耐久、碎屑、掉落与差量存档。

完成闸门：10 棵树 + 10 块离散岩石抽检无隐藏失效对象。

### S6 — 基础体验包

篝火：

- 聚焦燃料环、约剩余时间、E 快速添柴、长按上下文。
- 木柴/火焰/光/声音/UI 同步反馈。

背包：

- 快捷装备与库存分离。
- 工具、食水、医疗、材料、任务分区。
- 耐久、新鲜度、重量/堆叠和当前用途可见。

夜间：

- 黄昏/深夜/月光/黎明分段。
- 最低环境轮廓和眼适应。
- 手表背光、受限默认点火光源、可制作火把及雨/占手代价。

完成闸门：无营火时仍可近距导航；添柴不打开总制作菜单；常用装备不打开背包。

### S7 — 活动泡动物架构

交付：

- 区块种群账本继续负责远处出生、死亡、迁移和承载力。
- 玩家附近把账本槽位实例化为稳定个体。
- 个体有位置、生命、感知、目标、状态、受伤和短期记忆。
- 离开活动泡时把死亡、受伤、受惊和迁移折叠回账本。
- 对象数量、更新频率、寻路距离和降级策略有明确预算。

完成闸门：跟踪个体离开/返回视野不会无解释重置，远处区块不运行逐帧 AI。

### S8 — 动作战斗与首批生态

蛇：

- 盘伏、警告、瞄准、扑咬、恢复、受击、死亡。
- 几何命中与攻击时间窗；可抢攻、退让或绕行。

猎物：

- 觅食、饮水、警觉、逃跑、受伤、死亡、尸体处理。

捕食者：

- 追踪猎物/尸体、潜行/警告、惧火、攻击和撤退。
- 只在玩家已获得武器/治疗机会后进入遭遇导演。

生态回写：

- 捕猎、尸体、植被和水改变局部承载力/活动概率。
- 不是每个物种都同时激活；导演限制压力叠加。

完成闸门：不存在“进入半径即受伤”；每种威胁至少有两种可观察反制。

### S9 — 建造—群系—材料闭环

交付：

- 同类建筑多实例、多个营地、拆除与返还。
- 雨林/湿地/高地各至少一个具有独特规则价值的建筑。
- 首批加入柴架、接雨架/过滤架、晾晒/烟熏架、火把架、临时前哨、高脚平台中的最小组合。
- 材料来自对应群系，但旧群系材料仍能通过新能力产生新用途。
- 解锁由问题、材料观察和笔记推理触发，不靠线性任务发放全部答案。

完成闸门：测试者能解释为什么在不同群系建不同设施，且至少三座建筑改变世界规则。

### V1 — Valheim 启发的程序化视觉世界

专项依据：docs/VALHEIM_VISUAL_WORLD_STUDY.md

研究维度：

- 大地形轮廓与局部噪声的分层。
- 群系边界、植被簇、空地和视线节奏。
- 低多边形模型 + 有层次材质/顶点色。
- 太阳、天空、雾、天气、湿润度、色调和后处理。
- 粒子、风、音景与地标构图。
- LOD、实例化、阴影范围、对象池和包体预算。

交付：

- 先记录本机基线，再建立帧时间、draw calls、三角面、内存和包体预算。
- 视觉只消费稳定的群系/语义对象数据。
- 视觉差异帮助识别资源、天气、路线和危险。
- 原创热带雨林风格；不复制 Valheim 资产、色板、模型、文本或独特构图。

完成闸门：盲测玩家仅凭画面能区分群系和主要资源倾向；视觉升级不破坏交互轮廓和性能预算。

## 4. 动态团队组织

每个任务包只创建有明确交付和闸门的临时角色，不设置长期空泛岗位：

- 主协调/集成：维护 Brief、依赖、冲突边界、最终验收和用户沟通。
- 世界系统代理：S2/S3/S5，负责语义生成、对象能力和采集纵切。
- 持久化代理：S1，并对 S5/S7/S9 的差量状态做红队。
- 交互/UX 代理：S4/S6，负责聚焦、反馈、背包、篝火和夜间底线。
- 生态/战斗代理：S7/S8，负责具身动物、攻击窗口与种群回写。
- 程序化视觉代理：先只读研究，V1 闸门打开后实施。
- 可玩性红队：不写功能，从玩家路径、歧义、性能和重载一致性攻击每个候选版本。

所有代理共享以下约束：

- 不发布、不提交他人改动、不覆盖未归属文件。
- 每个改动附测试、可观察证据和已知限制。
- 一次只推进一个可玩的系统切片；禁止用大量内容掩盖未通过的闸门。

当前编码顺序：

1. 收口 G0.5 红队：篝火声画同钟、触屏制作/阻塞解释、暴雨点火一致、河水严格焦点遮挡、所有空间交互提交前同步最新玩家姿态。
2. 浏览器红队火焰威慑的边缘反馈与动物移动可读性，再推进下一条群系—材料—建筑中循环；已有拆除事务继续服从精确实例、可见损失和本地优先保存。
3. 对夜间、动物、烟熏架、树木、岩石、长局存档和跨区性能做浏览器可玩性红队，并记录 V0 帧时间/draw-call/内存基线。
4. 红队通过后先实施 V1A 跨活跃环实例池，再以版本化/存档决策闸门推进 V1B 连续世界场；随后增加原创热带树种/岩性轮廓、群落构图和天气感官层次。
