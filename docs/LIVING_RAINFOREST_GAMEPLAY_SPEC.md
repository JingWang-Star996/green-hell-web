# CANOPY：活体雨林玩法与交互规格

状态：当前里程碑的执行依据
日期：2026-07-14

## 1. 产品命题

CANOPY 不再以复刻《绿色地狱》的任务和内容为目标。它吸收的是“人在雨林中通过观察、试验、准备和承担后果而生存”的精神，并发展为原创的单机第一人称雨林生存建造游戏。

核心承诺：

> 雨林不是背景，而是一张会被玩家读取、利用、改变并反过来影响玩家的资源—栖息地—风险网络。

玩家在世界中看到的离散对象，必须与模拟中的真实对象一致。不能再出现“长得一样，一棵能砍、一棵不能砍；一块能捡、一块只是布景”的隐藏二分。

## 2. 当前问题不是内容少，而是世界不可信

现有实现将世界拆成了互不一致的几层：

1. 渲染器生成高密度的纯视觉树木、灌木和石块。
2. 模拟器另外生成少量可交互树和拾取资源。
3. 生态系统维护区块种群，再把个体投影成会移动的视觉模型，但个体没有生命、受击、逃跑、死亡和尸体。
4. 蛇等威胁仍主要由距离判定触发，而不是可观察、可预判、可攻击的动作实体。
5. 大量操作在菜单中结算，世界中的对象缺少状态和因果反馈。

结果是玩家无法从画面形成稳定规则，只能试错记忆“哪些模型是真的”。这会同时破坏探索、采集、战斗、建造和沉浸感。

## 3. 五条不可违背的设计法则

### 3.1 视觉可供性契约

- 相同轮廓、材质和尺度的对象，必须提供相同的基础动作。
- 差异应表现为工具、时间、耐久、风险和产出差异，而不是“一个有效、一个无效”。
- 若对象只是地形或微型地表杂物，必须与资源对象在轮廓、尺度或材质上明显区分。
- 锁定的动作必须说明原因和解决办法；禁止静默无效。

### 3.2 世界先反馈，UI 再解释

动作首先改变手中工具、目标模型、声音、粒子、掉落物或环境状态；UI 只补充数字、原因和下一步。不能仅靠消息框宣告“砍了树”“添了柴”“被蛇咬”。

### 3.3 一切离散对象来自一个权威生成源

同一棵树、同一块岩石、同一只动物只拥有一个稳定 ID、一份状态和一套规则。渲染器消费模拟对象，不得再自行生成外观相同的“假对象”。

### 3.4 生态是受限但真实的系统

不追求逐片叶子的科学模拟。采用“区块种群账本 + 玩家附近具身个体”的两层模型：远处低成本演化，近处才生成可感知、可战斗、可死亡的个体。

### 3.5 扩内容必须增加关系，不只增加数量

新物种、新材料和新建筑只有在改变至少两个既有系统时才加入。例如藤条既是雨林资源，也是捆扎材料、陷阱材料和某些动物的栖息条件；单纯新增一个换皮拾取物不算有效内容。

## 4. 统一世界对象协议

每个可见的离散世界对象都由同一个数据协议描述：

| 维度 | 必需内容 |
|---|---|
| 身份 | 稳定 ID、种类、物种/材质、生成版本、所属区块 |
| 形态 | 生长阶段、尺寸、朝向、碰撞体、视觉变体 |
| 状态 | 生命/耐久、资源余量、湿度、燃料、腐败或死亡阶段 |
| 动作 | 拾取、砍伐、挖掘、切割、攻击、检查、添加、拆除等能力 |
| 要求 | 工具类别、工具等级、体力、距离、角度、动作时间 |
| 反馈 | 高亮样式、动作文本、阻塞原因、命中反馈、状态变化 |
| 产出 | 确定性产出表、品质/数量修正、掉落方式 |
| 生命周期 | 再生、成长、迁移、腐败、拆解或永久改变 |
| 持久化 | 只保存相对世界种子的变化量，不重复保存未改变基线 |

这不是一个巨大的继承体系，而是数据驱动的组件组合。树、岩石、动物和建筑可以共享“可受击”“有库存”“会再生”“可燃烧”等能力。

## 5. 统一交互链

所有世界交互遵循同一条链：

1. 感知：距离、视线、准星锥和遮挡筛选候选对象。
2. 聚焦：只选一个主目标，给出克制的轮廓/边缘光和对象名称。
3. 解析动作：结合手中物品、目标状态和玩家状态决定主动作与备选动作。
4. 预告代价：显示所需工具、预计耗时、耐久或风险；未知产出不必提前剧透。
5. 执行动作：播放手部/工具动作，命中窗口才结算。
6. 命中反馈：目标形变或受击、材质匹配音效、碎屑、工具反冲。
7. 世界结果：扣减目标状态，生成掉落/尸体/残桩，改变火焰或建筑。
8. UI 解释：简短说明产出、失败原因或数值变化。
9. 持久化：只记录已改变的实体和关键进度。

基础按键语义：

- E：使用、拾取、打开、快速维护。
- 鼠标左键：手中工具/武器的主动作；对世界对象使用同一套命中规则。
- 长按 E：打开对象的上下文操作；不把全部行为重新塞回总菜单。
- 1–4 / 滚轮：切换装备；快捷栏与背包库存分离。

## 6. 世界对象改造清单

| 对象 | 当前矛盾 | 目标规则 | 优先级 |
|---|---|---|---|
| 树木 | 高密度视觉树与少量可砍树分离 | 所有离散树模型来自语义树；树种、年龄和直径决定工具、耗时、木料和倒伏结果 | P0 |
| 岩石 | 装饰岩石与可拾取石头难以区分 | 小型松散石可拾取；所有离散岩体可开采；连续地形岩壁是地形且外观明确不同 | P0 |
| 灌木/药草 | 普通灌木和资源植物轮廓接近 | 可收割植物有物种形态和生长阶段；微型地被不使用相同轮廓 | P0 |
| 动物 | 移动物主要是视觉投影 | 附近个体有感知、生命、行为、伤害、死亡、尸体和稳定 ID | P0 |
| 蛇 | 进入范围后判定事件 | 蛇是动作实体：盘伏、警告、瞄准、扑咬、恢复；可提前发现、攻击、击退或绕行 | P0 |
| 篝火 | 燃料只在菜单和文本中可见 | 聚焦显示燃料环/剩余时长；E 快速添一份合适燃料；火焰、木柴和光照同步变化 | P0 |
| 水体 | 规则可能绑定隐藏热点 | 看见并够得着的水面才可取水；水质由可观察环境和调查信息解释 | P1 |
| 掉落物 | 资源点与地表装饰混杂 | 掉落物有轻微近距高亮、清晰轮廓和一致拾取距离 | P1 |
| 建筑 | 自由放置后仍有菜单式维护残留 | 每座建筑拥有本地状态和上下文操作；支持多实例、拆除、维护和空间作用域 | P1 |
| 地标设施 | 模型尺度与交互锚可能分离 | 碰撞、交互锚和可见把手/面板来自同一几何契约 | P1 |
| 微型草叶/碎石 | 若全部交互会造成噪声 | 明确归为不可选中的地表“纹理层”，不得复用资源轮廓 | P2 |

## 7. 树木与岩石的趣味，不只是延长进度条

### 7.1 树木

- 树种影响木材用途：轻木适合火种和临建，硬木适合耐久构件，棕榈提供叶材和果实。
- 生长阶段影响动作：幼株可用刀切，细树可用石斧，巨木也能尝试砍伐但需要更好工具和更长时间。
- 错误工具仍产生可理解反馈，例如“石刃只能剥取纤维，无法有效切开这段硬木”，而不是没有反应。
- 命中位置和节奏影响效率，不改变总产出到失衡程度。
- 完成后至少经历树干倒伏/替代、树桩、木段/枝叶掉落；之后才进入缓慢再生。
- 砍伐降低局部遮蔽与植物承载力，短期增加可见度，长期影响小型动物活动。

### 7.2 岩石

- 小型河石：直接拾取，主要用于石刃和基础工具。
- 石灰/花岗质岩体：用石锤或鹤嘴工具开采建筑石料。
- 燧石结核：数量少，提供更锋利的切割材料。
- 黏土/红土露头：雨后更易开采，用于容器、炉灶和防雨构件。
- 岩体表面出现“裂隙点”；准确连续命中提高单位时间效率和反馈强度，但不凭空增加总资源。这借鉴的是 Rust 的可读命中节奏，而不是复制其美术或数值。
- 岩壁和山体本身属于连续地形，不显示可采集轮廓；所有单独摆放的岩石模型默认可开采。

Rust 官方资料把矿点分为石、硫、金属，并用发光热点给连续准确命中提供效率反馈；其地形/群系层还直接控制树、矿、食物与资源生成。这说明“视觉地貌—资源规则—动作反馈”应是一体设计，而不是装饰层与玩法层各自生成。

## 8. 具身动物与受限生态链

### 8.1 两层生态

远处区块保存低成本账本：

- 物种数量、承载力、出生/死亡、迁移和玩家捕猎压力。
- 群系、植被、水、天气、昼夜和尸体资源影响承载力。
- 不为远处每只动物逐帧寻路。

玩家附近的活动泡生成具身个体：

- 稳定 ID、生命、耐力、位置、感知、目标、行为状态和伤害状态。
- 离开活动泡后把死亡、受伤、受惊和迁移结果折叠回区块账本。
- 同一只动物在短期离开视野后不能无解释重置。

### 8.2 首批三种角色

1. 蛇：环境威胁与动作战斗教学；可观察、可绕行、可先手攻击。
2. 刺豚鼠/小型啮齿猎物：觅食、饮水、受惊、逃跑、被猎杀和尸体处理。
3. 中型捕食者：追踪猎物/尸体，惧火，只有在玩家具备长矛和治疗机会后进入导演池。

后续再加入食腐者、蛙类、鱼类和大型食草动物，避免先堆换皮生物。

### 8.3 行为状态

休息 → 觅食/饮水 → 警觉 → 逃跑/潜行 → 攻击 → 受击/硬直 → 死亡 → 尸体腐败/被食腐。

每个攻击必须有：

- 可定位的声音或姿态预兆。
- 攻击范围、方向和命中时间窗。
- 玩家可做的反制：退让、格挡、闪避、抢攻、照明或利用地形。
- 实际碰撞/射线结算，而不是单纯进入半径就扣血。

Nightingale 的开发者在 GDC 公开讨论了按地形偏好、体型差异和程序化种群填充来让世界“活起来”。CANOPY 采用同样的分层问题意识，但以浏览器预算实现更小、更确定的活动泡。

## 9. 背包、装备与上下文反馈

背包不再是一面平铺的物品墙：

- 一级分区：快捷装备、工具/武器、食物/水、医疗、材料、任务/记录。
- 快捷装备区与库存分开，直接显示当前手持、耐久和备用。
- 食物显示新鲜度阶段，工具显示耐久，材料显示最相关的已知用途。
- 默认按类别与紧急性排序；提供搜索/筛选，不要求玩家手工做像素级整理。
- 建筑材料允许“捆”或长材槽，避免大量相同长木棍挤占普通格。
- 世界拾取时显示去向、重量变化和超重后果。
- 聚焦世界对象时，背包只提示与当前动作相关的工具，不弹出整套制作菜单。

视觉上可以学习《森林之子》让库存具有“摊开野外装备”的空间感，但网页版本优先保证分类、状态和动作清晰，不机械复刻其摆放方式。

## 10. 夜间基础体验

黑暗仍是风险，但不能等于长时间失去玩法：

- 夜晚分为黄昏、深夜、月光恢复和黎明四段，完整不可读的深黑段缩短。
- 提供缓慢眼适应和最低环境轮廓，不把全局亮度直接拉成白天。
- 手表拥有短时背光，只照亮信息和极近处手部。
- 玩家默认拥有弱打火机/火镰光源：照射距离短、占用一只手、雨中不稳，保证最低导航。
- 可制作火把：范围更大，但消耗燃料、暴露位置、雨中衰减且影响单手/双手武器。
- 营火、火把架和遮雨灯成为营地与路线规划工具。
- 提供亮度/危险字幕等可访问性选项，但默认平衡仍由世界规则承担。

## 11. 篝火的局部交互标准

聚焦篝火时直接显示：

- 火焰状态、燃料环和约剩余时间。
- E：加入一份优先级最高的普通燃料。
- 长按 E：选择燃料、烹饪、煮水、取炭、熄灭或拆除。
- 阻塞原因，例如木材太湿、被暴雨浇灭、没有引火物。

添柴后同时发生：

1. 手部放入木料。
2. 木柴模型数量/焦黑程度变化。
3. 火焰、光照、火星和声音有短时增强。
4. 燃料环平滑上升。
5. 一条短消息说明增加了多少时间。

## 12. 建造由雨林问题驱动

建筑分类不再只是 UI 分页，而是解决不同群系与旅程阶段的问题：

| 问题/场景 | 建筑 | 关键材料 | 形成的循环 |
|---|---|---|---|
| 暴雨与湿物 | 斜顶叶棚、木柴架、排水沟 | 棕榈叶、藤条、轻木 | 找干料 → 保火 → 延长远征 |
| 水与容器 | 接雨架、黏土容器、过滤架 | 半椰壳、黏土、炭、砂石 | 采土/烧制 → 稳定净水 |
| 食物保存 | 晾晒架、烟熏架、封闭储物 | 硬木、藤条、火 | 狩猎 → 处理 → 储备 |
| 夜间路径 | 火把架、反光路标、临时避雨点 | 树脂、布/纤维、枝材 | 建立安全路线网络 |
| 动物压力 | 小型陷阱、抬高储物、简易围栏 | 弹性木、藤条、尖桩 | 观察兽径 → 布置 → 收获/防护 |
| 沼泽与洪水 | 高脚平台、栈道 | 硬木、藤条 | 进入新群系 → 获取独特材料 |
| 高地远征 | 挡风棚、石灶、储物桩 | 石料、硬木 | 建立前哨 → 扩张探索半径 |

技术要求：

- 同类建筑支持多实例和多个营地。
- 结构状态、作用半径、碰撞和维护都绑定具体实例。
- 建造预览继续由模拟器二次验证；加入拆除、返还和可理解的放置失败原因。
- 解锁来自遭遇问题、观察材料和笔记推理，不使用纯线性“任务发配方”。

## 13. 三层循环

### 微循环：10–90 秒

发现目标 → 选择工具 → 执行动作 → 获得材质匹配反馈与产出 → 决定携带或使用。

### 营地循环：10–40 分钟

识别天气/伤病/食物问题 → 远征收集 → 建造或维护设施 → 提升安全、储备和行动半径。

### 探索循环：1–3 小时及更长

进入新群系 → 学习新资源与生态风险 → 建前哨/路线 → 获得新能力 → 改变旧群系的利用方式。

“可玩十小时”不能靠十小时倒计时或重复刷新实现，而要让三个循环互相产生新计划。

## 14. 技术分层

1. World Definition：世界种子、群系场、语义生成规则。
2. Entity Registry：稳定 ID 与统一对象组件。
3. Capability Resolver：根据玩家、工具、目标和环境解析动作。
4. Deterministic Simulation：唯一结算生命、耐久、产出、生态和建筑。
5. Active Bubble：只激活附近动物 AI、碰撞和高频状态。
6. Renderer：只表现权威状态，可做动画预测但不能发明产出。
7. Save Delta：保存相对生成基线的改变；未改动世界可无限重建。
8. UI Projection：从模拟状态生成背包、提示、燃料环和日志。

关键迁移：删除“视觉树列表 + 可交互树列表”的双轨设计。区块生成先产生语义对象，再按距离和性能级别选择其表现精度。

## 15. 实施顺序与闸门

### M0：审计与契约（当前）

- 列出所有双轨对象和隐藏规则。
- 固化本规格、项目 Brief、试玩量表与架构测试。
- 收尾不与玩法冲突的存档 P1，暂停发布。

闸门：团队能对每类可见对象回答“它是什么、能做什么、为什么”。

### M1：世界真相与基础可读性

- 合并树/岩石/植物的语义与视觉生成。
- 实现统一聚焦、高亮、动作解析和阻塞原因。
- 完成岩石开采、树木分级砍伐、掉落与差量保存。
- 完成篝火局部反馈、背包分区、弱基础光源和火把。

闸门：玩家第一次看到任意离散树、石或资源植物时，九成目标能凭外观与聚焦反馈正确预测基础动作。

### M2：动作战斗与具身生态

- 蛇转为可攻击动作实体。
- 加入一种猎物与一种中型捕食者。
- 完成受击、硬直、逃跑、死亡、尸体处理和种群回写。

闸门：蛇不再能用“进入半径即结算”伤害玩家；每个威胁至少有两种可观察反制。

### M3：建造—群系—材料闭环

- 支持多实例建筑、拆除和多个营地。
- 加入首批问题导向建筑与群系材料。
- 让天气、生态、储存和路线共同制造建造理由。

闸门：至少三种建筑改变实际规则，玩家可描述为什么在不同群系建不同设施。

### M4：程序化视觉世界

- 在玩法实体与群系数据稳定后，依据 Valheim 专项研究升级地形层次、植被组织、材质、光照、雾、天气、色彩、音景、LOD 和性能策略。
- 保留原创热带雨林风格，不复制 Valheim 资产、色板、模型或构图。

闸门：视觉差异能帮助玩家读取群系、天气、资源和危险，而不只是截图更漂亮。

### M5：长循环与内容扩张

- 在前三个循环的留存和理解指标成立后，再扩物种、建筑、伤病、食谱、远征目标和叙事。
- GitHub、Toy 和外部发布只有在本地可玩闸门通过且用户确认后恢复。

## 16. 验收指标

- 同形同规则：抽查所有离散树、岩石和资源植物，不允许同模型随机失效。
- 首次理解：看到聚焦反馈后 5 秒内，玩家能说出主动作和缺失条件。
- 命中可信：攻击/采集只在工具命中窗口与目标碰撞时结算。
- 蛇反制：玩家可在被咬前发现并杀死、击退或绕开蛇。
- 动物真实性：跟踪一只动物 60 秒，可观察至少两次由环境/玩家引起的状态变化。
- 篝火反馈：添柴后 1 秒内至少有模型/火焰、声音和燃料 UI 三类反馈。
- 夜间底线：无营火时仍能用受限光源完成近距离导航和回营，不要求调高显示器亮度。
- 背包效率：常用装备切换不打开背包；查找任一类别物品不超过两次操作。
- 存档一致：砍伐、开采、动物死亡和建筑实例在重载后不复活/丢失，未改变区块不增加存档体积。
- 性能：视觉升级前先记录基线；任何新增活动泡系统都必须有明确对象上限和降级策略。
