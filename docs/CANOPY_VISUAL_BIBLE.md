## 0. 读法与边界

本文把“有限资源也能形成有层次的世界”翻译为 CANOPY 自己的热带雨林规则，不把另一个游戏的画面当目标图。所有条目使用以下证据标签：

- **[F]**：官方一手公开事实，或当前仓库中可定位、可重复的事实。
- **[I]**：由事实推导的工程判断；不是对其他游戏私有实现的声明。
- **[D]**：CANOPY 的原创设计选择或建议值；十六进制色值均为 sRGB 建议值。
- **[M]**：按固定 build、设备、seed、机位和路线采集的浏览器实测。本文不新增任何 [M]。

Valheim 在本文中的事实边界只有：其官方把世界描述为程序化生成；不同群系拥有不同环境、资源、敌人与玩法内容；官方 FAQ 描述了低分辨率纹理、稀疏多边形细节与现代材质、光照及后处理的结合；官方曾公开 Asset Bundles、软引用、引用计数及特定类别的动态装卸。连续噪声场、LOD 距离、实例池、shader 和构图算法没有公开证据，本文对应内容一律是 **[I]/[D]**。

### 玩家承诺 [D]

关闭 HUD 后，玩家仍应在两秒内回答四个问题：

1. 哪里能走，哪里会减速或阻断；
2. 哪个对象能做什么、当前处于什么状态；
3. 水、食物、药材、木材或石料更可能在哪个方向；
4. 天气和生态正在提示什么危险。

任何视觉差异必须至少服务 **交互、导航、生态、天气** 之一；仅“更丰富”而没有上述作用的差异不进入生产。氛围是这四项因果一致后的结果，不是第五个免责理由。

### 真值原则 [D]

- 离散树、岩石、植物、动物、尸体、建筑和资源线索必须来自语义/模拟对象；renderer 不得制造可交互替身。
- 微型地被、远冠层和气氛粒子必须明确 `never-focus`，且不得复用可采对象的尺度、轮廓、材质强调或局部运动。
- 轮廓、交互锚点、碰撞体、命中窗口、生命周期和存档 ID 必须描述同一个对象。
- 资源痕迹只有在对应资源/生态真值存在时才生成；尚未落地的鱼、芦苇、黏土等内容不得用“可收集外观”提前许诺。
- 低画质可减少表面密度，不得改变可交互 ID 集、基础动词、威胁可见性或夜间路线真值。

### 1.2 当前必须保留的事实边界 [F/I]

- `generateChunkDescriptor` 逐整数区块散列 elevation/moisture/canopy。V0 对默认 seed `1` 的 41×41 抽样同群系邻边仅 29.9085%，单格岛 32.0842%；这是棋盘感的代码证据。[F]
- 当前地形高度、固定河流与语义 `waterPresence` 仍是多套地理真值；本 Visual Bible 不授权偷偷替换生成器。连续世界场属于 V1B，需生成器版本与旧存档策略的用户 gate。[F]
- 当前通过的 V0 baseline architecture 是 `semantic-post-v1a-global-tree-rock-pools-v1`：树与岩石均为 `per-active-ring` 共享池，植物与 clutter 仍为 `per-nonempty-chunk`。[F] seed `1` 的当前静态源模型 known main / semantic draw inventory 为标准档 `92 / 58`、低端档 `40 / 26`；它们是 [I] 静态 inventory，不是浏览器 `renderer.info` 或 GPU [M]。
- `visualVariant` 已携带树种、年龄、岩性、植物与生命周期语义；V2 必须让它选择真实几何/组合，而不只是字符串或颜色。[F/D]
- 世界中所有离散树可处理，差异由工具、时间、耐久和产出表达；所有离散矿体可由当前石镐开采，大小表达耗时/耐力/磨损；小散石是拾取，微型 pebble 是 never-focus。[F]

### 1.3 交互视觉状态 [F/D]

视觉不得另造交互规则，只消费 `ResolvedAffordance`：

| 状态 | 当前交互模式 | 画面职责 [D] | 禁止 |
|---|---|---|---|
| `ready` | `execute` / `inspect` | 焦点对象局部提亮；动词落点和资源部位清楚 | 全对象常亮、用颜色替代轮廓 |
| `blocked` | `unavailable` | 保留对象身份；限制色只落在工具/部位提示，HUD 给出具体 blocker | 让对象像不存在，或仍显示“按 E” |
| `depleted` | `unavailable` | 用树桩、采空体、缺叶/新芽等物理状态表达 | 瞬间删除而无残迹，或继续显示成熟资源 |
| `ambient` | `unavailable` | 低对比、never-focus，服务地表/生态/天气 | 复用拾取堆、药草、矿脉或危险的轮廓 |
| `danger` | `execute` / `movement` | 先有生态预警，再有实体轮廓和攻击窗口 | 纯距离伤害、凭红光凭空标敌 |

焦点高亮仍是一时的确认反馈，不是世界可读性的主要来源。玩家未瞄准时也必须能凭 **形、尺度、姿态、状态部位** 预测基础动词。

## 2. 全局视觉语法

### 2.1 近—中—远三环 [D]

| 距离 | 必须保留 | 可删减 | 视觉问题 |
|---|---|---|---|
| 0–35m 近景 | 语义对象、生命周期、碰撞/命中、交互部位、地面通过性 | 非焦点微碎屑、非关键阴影 | “它是什么、能做什么、状态如何” |
| 35–90m 中景 | 树群边、林隙口、主要岩体、水带、地标体块、危险移动方向 | 小植物、细枝、局部动画与材质层 | “我往哪走、下一段会遇到什么” |
| 90–180m 远景 | 冠层高度带、山脊、水/雾走向、人造地标剪影 | 交互、碰撞、单株身份、透明细叶 | “我在哪里、目的地方向是什么” |

距离切换必须有滞回并由雾/遮挡掩护。远景代理永不进入 focus、碰撞、资源数量或存档；近景实体从模拟投影，不能由远景代理反推生成。

### 2.2 垂直层 [D]

每个可见方向至少形成三层中的两层，林隙/高地允许有意缺层：

1. **地表 0–0.6m**：根、落叶、泥水、散石与通过性；负责脚下风险和拾取尺度。
2. **下木 0.6–3m**：药草、灌木、藤环、幼树与危险遮挡；负责交互与近场压迫。
3. **树干/附生 3–12m**：板根、干柱、藤本和空隙；负责方向节奏与树种身份。
4. **冠层 12–30m**：冠层缺口、风雨响应和远景高度带；负责天气与宏观导航。

### 2.3 Value grouping 与形状语法 [D]

- 一帧只设一个主要明暗焦点：可走出口、资源机会、危险或地标；强调色面积通常不超过画面 5%。
- 前景最暗、对比最高；中景保留主色组；远景降低饱和度与细节但保持轮廓，不靠全屏灰雾抹平。
- 可走走廊用连续的地表 value 带表达；阻挡边界用根、树干、岩壁或深水形成垂直/高对比截断。
- 拾取石是低、平、成小堆；矿体是扎地的体块并有击打面；地形岩壁连续且没有“独立资源边缘”。
- 手采阔叶以叶脉十字、驱虫草以花冠、纤维藤以悬垂环、果灌木以果簇作为稳定地标；生长/耗尽改变这些部位的强度而不换基础动词。
- 树种/年龄可改变粗细、叉干、板根和冠幅；不得让同一剪影在一处可砍、另一处纯装饰。

## 4. 共享资产与运行时预算

预算服从 `VISUAL_MILESTONE_PERFORMANCE_PROTOCOL.md`，下表是 V2 的更窄资产合同。数字是 **[D] 上限**，不是当前 [M]；超预算即停止扩张，不以“Toy 还有 140MB”辩护。

### 4.1 共享几何、材质与实例

| 家族 | 当前每区块语义上限 [F] | 标准 5×5 / 低端 3×3 逻辑槽下限 | V2 共享 geometry 上限 [D] | 共享 material 上限 [D] | 阴影/焦点规则 |
|---|---:|---:|---:|---:|---|
| 树 | 17 | 425 / 153 | 9（站立构件与现有树桩/枝/原木状态合计） | 3（树皮、冠叶、切面/强调） | 近景关键/最近者可投影；中远景无逐株阴影；同一 focus 色覆盖，不克隆材质 |
| 岩石 | 16 | 400 / 144 | 5（完整体、破碎体、accent、rubble 合计） | 2（岩体、断面） | 仅近景/焦点候选投影；partial/exhausted 更新稳定 slot |
| 可采植物 | 17 | 425 / 153 | 4（叶、茎、物种地标、耗尽/新芽构件） | 2（植被、花/果/叶脉强调） | 默认不投影；高亮不新增 draw/material |
| micro-clutter | 34 | 850 / 306 | 4（不得因 V2 增长） | 1 | 永不投影、永不 focus；低端确定性裁减 |
| 中远冠层代理 | 非语义 | 由群落中心派生，不生成对象 ID | 4 | 2 | 无碰撞/交互/逐株阴影；只保留导航轮廓 |

“逻辑槽”是最大可见语义对象数，不等于给每个 variant、生命周期和 LOD 各乘一遍容量。物理 instance buffer 的数量、字节与 reserve 必须在 benchmark JSON 中另列；任何 variant 增加都不得让最大容量按变体数线性膨胀。

全局树池与岩石池均已通过 V1A 主线及红队验收，当前 baseline 固定为 `semantic-post-v1a-global-tree-rock-pools-v1`。预算比较以标准档 known main / semantic `92 / 58`、低端档 `40 / 26` 的静态 inventory 为架构参考；V2 仍须另采 production 浏览器 [M]，不得把这四个数字当作实测 draw call。

### 4.2 材质与纹理合同 [D]

- V2 新增 `THREE.Material` 对象总数 **≤6**；对象差异使用实例色、顶点色、UV 单元和共享 uniform，不为单对象克隆材质。
- V2 最多新增一张 512² RGBA8 植被/切面图集与一张 256² 单通道 wetness/mask；包含 mip 的 resident estimate 必须列入报告。单阶段 Toy 包增量仍 **≤4MB**。
- 叶片优先不透明几何或 `alphaTest` cutout；禁止大量 `transparent/blending` 叶层和双面过绘森林。
- roughness/wetness 是天气共享参数；不为“湿树”“湿石”另建一套材质对象。
- 可交互 accent 只标识稳定部位（叶脉、果簇、断面、击打面），不允许全物体霓虹描边常驻。
- V2 不新增每群系灯光。沿用一组方向光/环境光；局部动态无阴影灯的全场上限仍为 3，火把/营地按既有固定选择策略竞争名额。

### 4.3 性能硬门 [D]

标准档目标：1920×1080、DPR 1，frame median/p95/p99 ≤16.7/22/33ms，`render.calls` median/p95 ≤140/180，triangles typical/peak ≤250k/500k，shadow triangles ≤120k，resident texture estimate ≤64MB。低端档对应 1280×720、DPR 1，≤33.3/40/55ms，calls ≤100/120，triangles ≤150k/200k，shadow ≤40k 或关闭，texture ≤32MB。

V2 还必须相对通过的 V1A 同路线满足：frame p95 回退 ≤5%、draw p95 增长 ≤10%、JS heap 增长 ≤10%、Toy 包增量 ≤4MB；热身后无任意 `>50ms` Long Task。达不到绝对预算与相对预算中的任一项，停止加资产。

## 5. 低端降级与禁止项

### 5.1 低端档等价合同 [D]

低端档必须保留全部可交互对象 ID、碰撞、锚点、基础剪影族、资源/危险地标和生命周期。允许按以下顺序降级：

1. 有效 DPR；
2. 中远景与非焦点阴影；
3. 雨滴、飞虫、落叶等粒子密度；
4. micro-clutter 的确定性子集；
5. 35–90m 群落的个体密度，改用更少的共享簇代理；
6. 90–180m 冠层代理分段数和更新频率。

不得降级：0–35m 交互剪影、focus/blocked/depleted 状态、危险实体、可走/深水边界、夜间基础路线、建筑灯火状态。动态质量使用滞回；不得在阈值附近每秒往返跳变。

### 5.2 明确禁止

- 复制、描摹或提示模型重制 Valheim/其他游戏的资产、shader、配色、地图、音频、标志性构图或截图布局。
- 把 CANOPY 定义成“热带版 Valheim”，或用像素相似度作为审查目标。
- 用每区块独立随机色/密度假装群系，用均匀撒点假装生态。
- 用浓雾、全黑夜、bloom、AO 或暗角掩盖平坦、重复、同形异义和不可达交互。
- 在 V1B 用户 gate 前改变 generator identity、地形真值、河流真值或旧存档世界。
- 让 renderer 生成可采树、岩、药草、资源堆、动物/危险替身；远景代理、生态痕迹和 ambient 也不得进入命令真值。
- 为每对象创建材质、光源、阴影或 draw；不得让 variant 数量乘实例容量。
- 用 4K PBR、高模堆积、SSR、SSAO、重体积光或大量透明叶片作为第一轮解决方案。
- 低画质隐藏可交互物或将药草、石料、危险退化成无法辨识的同一 primitive。
- 视觉状态领先或滞后模拟：例如采空岩仍完整、未点燃火把发光、雨被遮挡但地面仍持续 splash。

## 7. V2 单走廊最小切片

### 唯一推荐：E-01「常绿密林·板根林缘 → 药草林隙」

这是下一条且仅此一条 V2 走廊。选择它是因为它能在 **一个 v1 区块、一个固定 seed、现有语义对象** 内同时验证树、岩、植物、生命周期和三距离构图，不需要偷渡 V1B 的连续世界/旧存档迁移，也不扩写五群系资产。

#### 固定地址 [F]

- world seed：`1`；generator：v1；chunk：`0:-1`；当前 descriptor 为 `evergreen-rainforest`。
- 眼平路线：C05 `(39,-31)` → C04 `(29,-23)` → close `(24,-38)`，允许为避碰做 ≤2m 的固定偏移；路线与 yaw 必须写入 fixture，不能每次手调。
- 年龄对照：`semantic.tree.1:0:-1:2`（young ironwood，约 `22.8,-15.8`）与 `semantic.tree.1:0:-1:11`（old-growth ironwood，约 `24.5,-30.4`）。
- 岩体对照：`semantic.mineable-rock.1:0:-1:2`（granite / `granite-fractured`，约 `22.6,-31.4`）；用同一对象的 full 与 partial save fixture 对照，不复制第二个假矿体。
- 植物对照：`semantic.harvestable-plant.1:0:-1:4`（young medicinal-broadleaf，约 `24.2,-40.3`）与 `semantic.harvestable-plant.1:0:-1:3`（mature，约 `43.1,-23.0`）。

若这些 ID/地址因有意升级 generator 而变化，先升级 fixture 协议和存档决策记录；不得临时换一组更上镜的对象。

#### 只做这些 [D]

1. 铁木 `ironwood-buttress` 的 young / old-growth 两个年龄剪影：共享板根、干柱和冠团构件，年龄改变板根展开、干柱粗细、叉点与冠宽，不改变“装备斧→砍伐”的基础动词。
2. granite 的 intact / partial 两个破碎状态：partial 减少主体体量并露出稳定断面/碎屑，保持同一个 ID、锚点和可开采读法；exhausted 使用已有采空残迹，不新增战利品承诺。
3. medicinal-broadleaf 的 young / mature 两个生长态：共享淡叶脉十字，young 更低更疏，mature 更展开；full/partial/depleted/regrowing 继续控制资源部位强度。
4. 用现有对象构成“暗树柱入口—板根收窄—亮药草林隙”三段 value 节奏；不得移动语义对象来摆拍，不新增不可交互资源替身。
5. 仅使用第 4 节预算内的共享几何/材质；不在本切片增加新天气、后处理、世界场或其他群系资产。

| 语义等价 | before/after 的五个目标 ID、坐标、quantity、工具要求、交互锚、碰撞、tree/rock/plant lifecycle 与存档 round-trip 完全一致 |

| 构图 | C04/C05/close 三帧都有近—中—远；入口、收窄、林隙依次可读；不出现连续三株完全相同组合 |
| 交互 UI | ready/blocked/depleted 的 focus 色只确认真实 affordance；未瞄准也可预测基础动词，低端档不丢对象 |
| 性能 | 相对通过的 V1A 同路线 frame p95 ≤+5%、draw p95 ≤+10%、heap ≤+10%、Toy ≤+4MB；同时满足第 4.3 节绝对门 |
| 回归 | 20×12 contact sheet 无新增 P0；C08–C11 夜雨底线不退化；十区块往返无整块 pop/错位/`>50ms` Long Task |

#### 停止条件

出现任一项即回退：需要移动/复制语义对象才形成画面；近景剪影与 collider/anchor 不一致；每对象独立材质/draw；树池串位或存档身份变化；资源 cue 领先模拟；浏览器基线不稳定；性能超门；两种审美方向无客观胜负。后者返回用户做视觉 gate，不同时扩散两套风格。
