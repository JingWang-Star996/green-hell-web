# CANOPY：群系—材料—建筑中循环

日期：2026-07-14
状态：设计门槛完成；排在动作事务与通用尸体之后实施

## 1. 目标

下一批建筑不以“菜单里多三张配方”为目标，而要把当前已经存在的雨、夜晚、捕食者、潮湿、寄生虫、树木、岩石、狩猎和动态区块连接成可重复的中循环：

`进入新群系 → 遭遇可观察问题 → 发现当地材料/规则 → 在世界中选址建造 → 建筑改变局部规则 → 形成更远的探索半径 → 维护、拆迁或升级网络`

最小组合只做三座建筑：雨水收集架、火把路标、架高前哨。它们分别解决持续饮水、夜间路线/捕食者压力和湿地远征落脚点，不替代营火、叶棚、床和烟熏架。

## 2. 三座建筑

### 2.1 雨水收集架

**问题与群系**

- 河谷湿地、黑水沼泽降雨多，但直接饮水有寄生虫风险。
- 密林冠层会遮挡部分降雨；林隙和河岸是更好的选址。
- 当前“打开制作菜单立即接雨”应降级为应急手段；长期供水由世界建筑承担。

**复用材料**

- 木棍 ×6、藤蔓 ×4、宽叶 ×6、椰壳 ×2。
- 不新增水物品，产出继续使用现有 `clean-water`；椰壳仍决定玩家能携带多少。

**世界规则**

- 结构状态只有 `storedUnits`、`exposure`、`lastAdvancedTick`。
- 以一秒或更低频率聚合雨量，不逐滴模拟。有效速率由雨势、天空暴露、冠层和群系决定。
- 容量 4 单位；靠近后可一次收取背包能容纳的数量，满包不清空储水。
- 干旱时保留库存；暴风不额外凭空产水，只提高已存在降雨的速率。
- HUD 聚焦显示 `2/4`、当前收集速度和“被冠层遮挡/容量已满”等阻塞原因。

**解锁**

- 玩家在雨中手动用椰壳接水两次，或携带空椰壳观察一处明显林隙后，在笔记中推导配方。
- 不由主线任务直接发放。

### 2.2 火把路标

**问题与群系**

- 密林视距短，夜间即使有手表底光也难以维持路线记忆。
- 高地风口适合作为远距导航点，但暴雨会缩短燃烧；密林中的稀疏路线网络更有价值。
- 捕食者应能观察并回避火，而不是只在数值上降低遭遇概率。

**复用材料**

- 木棍 ×4、石块 ×3、藤蔓 ×1、火把 ×1。
- 不新增树脂。火把自身已有耐久段，放入路标后转为结构燃料秒数；拆除不能复制回一支满耐久火把。

**世界规则**

- 交互包含“点燃/熄灭”“补入火把”“取回未燃尽余量”三个上下文动作，默认 E 执行当前最合理动作。
- 只为玩家最近的 3 个点亮路标启用真实局部灯；更远路标使用 emissive、粒子和地图记号，避免灯光线性增加。
- 6 米内为基础可读区，8 米内给捕食者火焰恐惧输入；降雨与无遮蔽状态提高燃料消耗。
- 地图只记录玩家亲自点亮过的路标；熄灭后仍保留暗色标记，形成路线记忆而不是 GPS。

**解锁**

- 玩家在夜间让第一支火把消耗至少一个燃料段，或第一次观察到动物对营火退避后解锁。

### 2.3 架高前哨

**问题与群系**

- 黑水沼泽的地表潮湿、寄生虫和低视距会让长距离探索变成只靠背包数值硬扛。
- 架高前哨提供一小块干燥、可睡、可观察的立体安全面，但不等于永久无敌屋。
- 高地石料、密林原木、棕榈叶与狩猎兽皮形成跨群系准备，而不是单一区域原地毕业。

**复用材料**

- 原木 ×8、木棍 ×10、藤蔓 ×8、宽叶 ×6、石块 ×8、兽皮 ×2。
- 第一版不新增木板、钉子或绳索物品；若以后引入加工台，再把原木拆成构件。

**世界规则**

- 由四根桩、平台、梯坡和小顶棚组成；碰撞与可站立面来自同一共享布局。
- 顶棚与平台范围内降低雨淋和地表寄生虫风险，可放置一张床、一个烟熏架或一座雨水架；营火只能放在指定通风边缘，不能穿过平台。
- 提供一次性周边区块观察加成和本地前哨地图标记，不自动揭开无限地图。
- 睡眠仍推进饥渴、天气、火、腐坏和动物系统；捕食者可在平台外徘徊，受伤或没有床时不能把它当免费快进按钮。

**解锁**

- 玩家在黑水沼泽达到高湿度或感染寄生虫，并已经观察过原木、兽皮和高地石料后，通过笔记推导。

## 3. 建筑菜单与世界交互

建筑菜单按用途而不是按开发顺序分组：

- 营地核心：营火、叶棚、床。
- 水与食物：雨水收集架、烟熏架。
- 路线与安全：火把路标、架高前哨、求救信标。

每张建筑卡必须显示：解决的问题、放置约束、主要维护成本、群系修正和完整材料。开始放置后菜单关闭，世界预览给出绿色可放、橙色低效率、红色阻塞三态；旋转、取消、确认继续走现有唯一放置路径。

聚焦建成建筑时显示局部状态，不打开总制作菜单：

- E：最常用的快速动作。
- 长按/次级键：打开该建筑的上下文轮盘。
- 拆除属于上下文动作，需要短确认和可见返还预览。

## 4. 自由建造、拆除与返还

- 三座建筑均支持多实例和多个前哨，不与“营地中心”绑定。
- 每座建筑保存位置、朝向、建造时刻、耐久和最小运行态；效果只查询实际几何范围。
- 拆除返还基础材料的 65%；未消耗的容器或库存原物返还，已燃烧火把只按剩余耐久折算，处理中的食物保持真实到期时间。
- 战斗或天气损坏低于 50% 时返还再下降，避免无成本瞬移材料。
- 放置、加载、维护、拆除全部采用“预检 → 表现时间 → 二次校验 → 原子结算”，与当前制作原子性一致。

## 5. 存档与动态世界

当前 `camp.structures` 适合少量首夜建筑，但十小时、多前哨和无限区块需要转为按区块索引的稀疏玩家建筑：

```text
world.playerStructures[id] =
  [chunkX, chunkZ, localX, localZ, yawQ, kind, conditionQ, stateTuple]
```

- 稳定 ID 由 run epoch + 单调建造序号生成，不能由当前位置重新散列。
- 活动泡只物化邻近区块；离开时结构状态压回紧凑 tuple。
- 未改变的程序化世界仍由种子重建，玩家建筑是明确差量。
- 迁移时把现有 `camp.structures` 幂等搬入新表，保留位置、朝向、过程和建造时刻。
- 云存档压力测试至少覆盖 10 小时、80 座路标、12 个雨水架和 6 个前哨，仍满足 Toy 128 键与每物理值 1024 UTF-8 bytes 约束。

## 6. 十小时节奏

| 阶段 | 主要问题 | 建筑作用 | 循环变化 |
|---|---|---|---|
| 0–1 小时 | 首夜伤口、水、火、遮雨 | 现有营地核心 | 学会基础规则，不提前塞满建筑菜单 |
| 1–3 小时 | 雨天饮水与第一次远离营地 | 雨水收集架 | 从手动接雨转为选址、返回收取和容量管理 |
| 2–5 小时 | 夜路、迷失、首次捕食者 | 火把路标 | 收集石/木、布设路线、维护燃料、观察动物反应 |
| 4–7 小时 | 跨群系原木、兽皮与高地石料 | 前哨准备 | 树木、狩猎、采矿和运输形成中期目标 |
| 6–10 小时 | 黑水湿地与更远动态区块 | 架高前哨网络 | 建立第二落脚点、局部生产和多路线返程 |

时间只是内容排布预算，不承诺用等待或重复采集强行填满十小时。每个阶段必须至少引入一种新决策：选址、维护、路线、跨群系运输或风险交换。

## 7. 性能预算

- 雨水与燃料过程以 1 Hz 或离线聚合推进，不新增逐帧 React 状态。
- 同类小建筑共享几何/材质与活动环实例池；状态变化只更新 slot。
- 真实 PointLight 最多同时 3 个，按玩家距离和可见性选择；其余路标无动态阴影。
- 前哨模块共享柱、平台、顶棚几何；50 座建筑压力场景总 draw-call 增量目标不超过 12。
- 跨区挂载每帧预算 ≤4 ms，禁止一次性重建全部玩家建筑。
- 新增代码原生资产优先；确需纹理时使用一张 256–512px 原创图集，不按建筑复制材质。

## 8. 验收门槛

自动化：

1. 三种建筑的放置、旋转、碰撞、遮雨/灯光/平台范围共享同一几何真相。
2. 材料不足、位置变化、过程死亡、背包满、容量满、燃料满和拆除中断均为零物品副作用。
3. 雨势、冠层、群系和离线时间确定性推进雨水；同 seed/输入可重放。
4. 路标只激活最近三盏真实灯，火焰恐惧只作用于局部具身动物。
5. 前哨不会让营火穿模、床悬空或平台外仍获得遮雨/寄生虫保护。
6. 建筑离开活动泡、返回、手动存档、自动存档和 Toy 云往返后状态一致。
7. 十小时结构压力存档仍在 Toy 配额内；80 路标场景不线性创建灯或独立材质。

浏览器：

- 玩家不读教程也能从模型轮廓判断收集架、路标和前哨的大致用途。
- 固定路线覆盖林隙接雨、密林夜路、暴雨熄灯、捕食者惧火、沼泽前哨休息和拆除返还。
- 建筑菜单、世界提示和动作回执不互相遮挡；键鼠和触控完成同一条路径。
- 固定 1080p/720p 场景记录 frame median/p95/p99、draw、triangle、内存和跨区尖峰；没有浏览器数据时不得宣称性能门槛通过。
