# CANOPY：砍树—倒伏—原木纵切规格

日期：2026-07-14
状态：只读设计完成；排在动作蛇之后实施

## 1. 要解决的问题

当前树木的 `quantity` 同时承担耐久、掉落数量和存在状态。结果是每砍一下直接把木棍塞进背包，最后一击让树瞬间缩成残桩。这个结算没有倒伏、掉落、选择和后续加工，既不像砍树，也无法支撑建造材料循环。

下一纵切必须形成：

`砍伐 → 倒伏 → 拾枝 → 分段 → 搬取原木 → 劈分/建造`

## 2. 状态模型

站立树继续用 `quantity` 表示结构耐久；归零时生成父树内部的稀疏倒木状态：

```ts
interface TreeHarvestState {
  fallDirection: number; // 0..1023 量化角度
  branches: number;
  trunkSegments: number;
  looseLog: boolean;     // 同时最多存在一根待拾取原木
}
```

倒木材料不能生成大量独立 `customEntities`。枝条、树干段和待拾原木都保存在父树 delta 中，避免长局探索让 Toy 云存档线性膨胀。

旧存档中 `quantity=0` 且没有 `treeHarvest` 的树解释为已经处理完的树桩，不能补发材料。

## 3. 状态机

| 阶段 | 条件 | 主动作 | 结果 |
|---|---|---|---|
| 站立 | `quantity > 0` | 砍伐 | 扣结构耐久；不直接发材料 |
| 倒伏有枝 | `branches > 0` | 拾取枝条 | 每次获得 1 木棍 |
| 倒伏待分段 | `branches=0 && trunkSegments>0 && !looseLog` | 分段倒木 | 消耗时间、体力和斧耐久；产生 1 根待拾原木 |
| 倒伏有原木 | `looseLog` | 搬取原木 | 背包增加 1 原木 |
| 树桩 | 所有材料取尽 | 无 | 永久保留小树桩 |

最后一击的倒伏方向取“远离玩家”的方向，并量化后保存；玩家与树中心重合时由对象 ID 哈希决定。MVP 不加入刚体、压伤、连锁倒塌或树干物理模拟。

## 4. 基础产出

| 成长阶段 | 枝条 | 原木 |
|---|---:|---:|
| 树苗 | 1 | 0 |
| 幼树 | 2 | 1 |
| 成熟树 | 3 | 2 |
| 古树 | 4 | 3 |

棕榈可额外增加 1 枝条。新增 `log`，堆叠上限建议 6；新增“劈分原木”：1 原木，经斧具加工得到 3 木棍并损耗耐久。它把砍树闭环接回营火和首批建筑，但不在本纵切扩展完整建造树。

## 5. 模拟与存档约束

- 倒伏但仍有材料的树不能简单等同于 `depleted=true`。
- 世界 delta 升至 v2，同时兼容读取 v1。
- 处理完的树桩继续保持紧凑的零数量 delta。
- 背包满、工具损坏或命中失败时不能吞掉世界材料。
- 低画质只能减少环境杂物，不能隐藏倒木材料。
- 状态签名必须包含方向、枝条、树干段和 `looseLog`。

## 6. 交互与画面

交互锚点必须由模拟、准星、射线和高亮共享：

- 站立：树干砍痕。
- 拾枝：倒下的树冠端。
- 分段：下一处切口。
- 原木：当前待搬取木段。

表现层：

- 普通砍击：砍痕加深、短促摇晃、木屑和材质音。
- 最后一击：约 0.9 秒倒伏动画；模拟立即进入倒木状态，动画不入存档。
- 重新加载已经倒伏的树直接显示最终姿态，不重播动画。
- 倒木树干、枝条堆、原木和树桩继续走实例池，不增加逐树 draw call。

碰撞从站立圆形切换为倒伏胶囊/线段碰撞；树干处理完后只保留小树桩圆形。枝条和可搬原木不阻挡移动。

## 7. 验收

- 普通砍击不再直接获得木棍。
- 最后一击确定性倒伏并留下材料和树桩。
- 背包满、工具损坏时不吞材料。
- 高级树种工具要求不回退。
- 离开区块、重进、保存重载后方向和剩余材料一致。
- 旧存档不会重复生成原木。
- 相似外观的离散树全部遵守相同规则，不再区分“装饰树/资源树”。
- 连续处理 30 棵树不增加逐树 draw call，不生成大量自定义实体。
- 完整流程可走通：砍伐 → 倒伏 → 拾枝 → 分段 → 拾原木 → 劈分为木棍。
